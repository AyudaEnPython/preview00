Virtual Environments

Section titled “Virtual Environments”

Python applications often use packages and modules that are not part of the standard library. These applications may require a specific version of a library, and it may not be possible for a single installation to meet all requirements.

The solution is to create a virtual environment: a directory that contains a specific Python installation along with additional packages.

Package Manager

Section titled “Package Manager”

pip

Section titled “pip”

pip is a package manager that allows you to install, update, and uninstall Python packages from the Python Package Index (PyPI).

It includes commands such as install, uninstall, freeze, etc (see the guide for more information).

Note

On Windows use python -m pip ... and on macOS/Linux use python3 -m pip ...

python -m pip install installs the latest versio or a specific version of a package:

  • Latest
  • Specific

python -m pip uninstall followed by one or more package names, removes packages from the virtual environment.

python -m pip show displays information about a specific environment.

python -m pip list shows all installed packages in the virtual environment.

python -m pip freeze > requirements.txt save the list of installed packages into a requirements.txt file.

python -m pip install -r requirements.txt installs the packages listed in requirements.txt.

Version Control

Section titled “Version Control”

Version control is the practice of tracking and managing changes to code. A version control system keeps a history of all changes made to the code, including who made them, when they were made, and what changes were introduced.

Git

Section titled “Git”

Git has become the global standard for version control. It is a distributed version control system, which means that a local clone of a project is a complete version control repository. 2
