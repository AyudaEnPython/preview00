Entornos Virtuales

Sección titulada «Entornos Virtuales»

Usualmente las aplicaciones de Python hacen uso de paquetes y módulos que no forman parte de la biblioteca estándar. Estas aplicaciones a veces necesitan una versión específica de una biblioteca y tal vez no sea posible para una instalación cumplir con los requerimientos de todas las aplicaciones.

La solución a este problema es crear un entorno virtual, un directorio que contiene una instalación de Python de un versión en particular, además de unos cuantos paquetes adicionales.

Administrador de paquetes

Sección titulada «Administrador de paquetes»

pip

Sección titulada «pip»

pip es un administrador de paquetes que permite instalar, actualizar y desinstalar paquetes de Pytho desde el índice de Paquetes de Python (PyPI).

pip tiene varios subcomandos: install, uninstall, freeze, etc (consultar la guía para mayor información).

Nota

Para Windows usar python -m pip ... y para macOS/Linux python3 -m pip ...

python -m pip install instala la última versión (más reciente) o una versión especifica de un paquete:

  • Más reciente
  • Específica

python -m pip uninstall seguido de uno o varios nombres de paquete eliminará los paquetes del entorno virtual.

python -m pip show mostrará información de un paquete en particular.

python -m pip list mostrará todos los paquetes instalados en el entorno virtual.

python -m pip freeze > requirements.txt guarda en el archivo requirements.txt la lista de paquetes instalados.

python -m pip install -r requirements.txt instala la lista de paquetes presentes en requirements.txt.

Control de versiones

Sección titulada «Control de versiones»

El control de versiones es la práctica de rastrear y gestionar los cambios en el código. Un sistema de control de versiones guarda un historial de todos los cambios realizados en el código, incluyendo quién hizo los cambios, cuándo se hicieron y qué cambios se realizaron.

Git

Sección titulada «Git»

Git se ha convertido en el estándar mundial para el control de versiones. Es un sistema de control de versiones distribuido, lo que significa que un clon local del proyecto es un repositorio de control de versiones completo. 2
