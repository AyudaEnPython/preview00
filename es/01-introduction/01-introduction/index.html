¿Qué es Python?

Python 1 es un lenguaje de programación interpretado cuya filosofía hace hincapié en la legibilidad de su código. Se trata de un lenguaje de programación multiparadigma 2, ya que soporta parcialmente la orientación a objetos, programación imperativa y, en menor medida, programación funcional. Es un lenguaje interpretado, dinámico y multiplataforma.

Es un lenguaje de programación potente, fácil de aprender, de sintaxis elegante, tipado dinámico y fuerte que junto a su naturaleza interpretada lo convierten en el lenguaje ideal para scripting y desarrollo rápido de aplicaciones en muchas áreas, para la mayoría de plataformas.

Opcional: usa un script y obtén las fechas
import asyncio import re from pyodide.http import pyfetch async def get_release_date(version: str) -> str | None: BASE = "https://www.python.org" slug = version.replace(".", "") target = f"{BASE}/downloads/release/python-{slug}0/" url = f"https://thingproxy.freeboard.io/fetch/{target}" try: response = await fetch( url, method="GET", headers={"Accept-Encoding": "gzip"}, ) data = await response.bytes() content = data.decode("utf-8") pattern = re.compile( r'<header class="article-header">.*?' r"<p>.*?<strong>Release Date:</strong>s*(.*?)<", re.DOTALL, ) match = pattern.search(content) if match: return match.group(1).strip() except Exception: pass async def main(): for version in ("2.7", *[f"3.{x}" for x in range(8, 14)]): date = await get_release_date(version) print(f"Python {version.ljust(5)}| Fecha: {date}") await asyncio.sleep(0.5) if __name__ == "__main__": asyncio.run(main())

Nota

La ultima versión de Python 2.x es la 2.7 y a partir de la versión 3.8, cada versión estable se libera en el mes de octubre (PEP 602 3).

Implementaciones

Sección titulada «Implementaciones»

Existen varias implementaciones alternativas de Python que pueden ser de particular interés para diferentes audiencias:

ImplementaciónDescripción
CPythonEscrita en C, es la implementación original y la más mantenida. Las nuevas características del lenguaje aparecen aquí.
BrythonEscrita en JavaScript, permite la ejecución de código Python en navegadores web.
IronPythonEscrita en C#, diseñada para la plataforma .NET.
JythonEscrita en Java, permite la ejecución de código Python en la máquina virtual de Java (JVM).
MicroPythonEscrita en C, está optimizada para ejecutarse en microcontroladores.
PyPyEscrita en Python, utiliza un compilador JIT (just-in-time) para optimizar la ejecución del código, ofreciendo mejoras de rendimiento.

El Zen de Python

Sección titulada «El Zen de Python»

El “Zen de Python” es una colección de 20 principios de software, de los cuales 19 fueron escritos por Tim Peters 4 (1 reservado para Guido), originalmente posteado en un hilo llamado “The Way of Python”. 5

Está escrito como la entrada informativa número 20 de las “Propuestas de Mejoras de Python” (PEP, Python Enhancement Proposals). 6

  • Bello es mejor que feo.
  • Explícito es mejor que implícito.
  • Simple es mejor que complejo.
  • Complejo es mejor que complicado.
  • Plano es mejor que anidado.
  • Disperso es mejor que denso.
  • La legibilidad cuenta.
  • Los casos especiales no son lo suficientemente especiales para romper las reglas.
  • A pesar que lo práctico vence a la pureza.
  • Los errores nunca deberían pasar silenciosamente.
  • A menos que hayan sido silenciados explícitamente.
  • Frente a la ambigüedad, rechaza la tentación de adivinar.
  • Debería haber una -y preferiblemente solo una- manera obvia de hacerlo.
  • Aunque esa forma no esa obvia al principio a menos que seas holandés.
  • Ahora es mejor que nunca.
  • Aunque nunca es a menudo mejor que ahora mismo.
  • Si la implementación es difícil de explicar, es una mala idea.
  • Si la implementación es fácil de explicar, puede ser una buena idea.
  • Los espacios de nombres son una gran idea — hagamos más de esos.
