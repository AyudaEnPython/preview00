Algoritmo

¿Cómo las aplicaciones resuelven problemas y realizan tareas?

Objetivos

  • Comprender el concepto de algoritmo.
  • Identificar sus características y su composición.
  • Representar e interpretar algoritmos.

Definición

Sección titulada «Definición»

En matemáticas, lógica, ciencias de la computación y disciplinas relacionadas, un algoritmo es un conjunto finito, ordenado y explícito de instrucciones o reglas definidas y no-ambiguas que permite, típicamente, solucionar un problema, procesar datos o realizar un algún cálculo. 1

Se puede comparar el concepto de algoritmo con una receta de cocina. Así, los algoritmos habitualmente están formados por secuencias de instrucciones que probablemente se repitan (iteraciones) o que requieran decisiones (comparaciones lógicas) hasta que completen su tarea.

block-beta A["Algoritmo para instalar una nueva dependencia"]

  1. Situarse en el directorio raíz del proyecto.

  2. Activar el entorno virtual

  3. Instalar la nueva dependencia.

  4. Verificar instalación

  5. Actualizar el archivo de requerimientos.

Cuando un algoritmo está asociado con el procesamiento de información, los datos se leen de una fuente o dispositivo de entrada, se procesan y emiten un resultado por un dispositivo de salida, o bien se almacenan en otro dispositivo para su uso posterior.

Resumen

Conjunto ordenado y finito de operaciones (pasos).

Características

Sección titulada «Características»
mindmap root[Algoritmo] Preciso Definido Finito

Todo algoritmo debe ser:

  • Preciso: Debe especificar sin ambigüedad el orden en que se deben ejecutar todas las instrucciones.
  • Definido: Cada vez que se ejecute bajo las mismas condiciones arrojará el mismo resultado. La secuencia de ejecución deberá ser la misma.
  • Finito: Debe terminar en un número finito de pasos, en un tiempo finito y requiriendo una cantidad finita de esfuerzo.

Composición

Sección titulada «Composición»
flowchart LR subgraph DataFlow["Anatomía de un algoritmo"] direction LR Entrada@{ shape: in-out } Entrada:::io --> Proceso Proceso:::process Proceso --> Salida@{ shape: in-out } Salida:::io end classDef io fill:#bbdefb,stroke:#0d47a1,stroke-width:2px; classDef process fill:#c8e6c9,stroke:#1b5e20,stroke-width:2px,stroke-dasharray:4 2;
  • Entrada: Está conformada por la información dada al algoritmo o conjunto de instrucciones que generan los valores con los que se ha de trabajar en caso de que carezca de datos de entrada.
  • Proceso: Lo conforman los cálculos y operaciones necesarias para que a partir de los datos o instrucciones de entrada se llegue a los resultados.
  • Salida: Son las transformaciones que ha sufrido la información de entrada a través del proceso para obtener los resultados finales.

Representación

Sección titulada «Representación»

Pseudocódigo

Sección titulada «Pseudocódigo»

Es una descripción de alto nivel de un algoritmo que combina lenguaje natural con algunas convenciones sintácticas pero sin ceñirse a un estándar fijo.

Precaución

No existe una sintaxis estándar para el pseudocódigo; sin embargo, existen ciertos estándares limitados (por ejemplo, para evaluaciones académicas):

Está diseñado para que cualquier persona entienda la lógica central y, al mismo tiempo, sea lo suficientemente estructurado como para traducirse directamente a un lenguaje de programación. 2

En general, es comprensible y estructurado para que su implementación se pueda hacer directamente a partir de él sin importar el lenguaje de programación.

Aquí tienes un ejemplo del pseudocódigo anterior en diferentes lenguajes:

  • Python
  • JavaScript
  • C
  • C++
  • Rust
  • Golang
def main(): x = int(input("First number: ")) y = int(input("Second number: ")) result = x + y print(f"Result: {result}") if __name__ == "__main__": main()

Resumen

Facila a las personas el entendimiento de un algoritmo.
